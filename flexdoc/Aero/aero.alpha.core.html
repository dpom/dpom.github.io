Public Var Description
eval-tagged-literal Dispatches a tagged literal with control over eval. Dispatch happens on the :tag of the tagged-literal. opts are the options passed to aero.core/read-config. env is a map of already resolved parts of the config. ks is a vector of keys which make up the current position of the tagged literal.
expand Expand value x. Dispatches on whether it's a scalar or collection. If it's a collection it will expand the elements of the collection.
expand-case Expands a case-like value, in the same way as #profile, #user, etc. case-value is the value to dispatch on, e.g. the result of (System/getenv "USER") for #user. tl is the tagged-literal where the :value is a map to do the casing on. See implementation of #profile for an example of using this function from eval-tagged-literal.
expand-coll Expand value x as a collection. Does not work with non-collection values.
expand-scalar Expand value x without expanding any elements it may have. Takes either a scalar or a collection (which will be treated as a scalar).
expand-scalar-repeatedly Expand value x until it is either incomplete or no longer a tagged-literal. Use this to support chained tagged literals, e.g. #or #profile {:dev [1 2] :prod [2 3]}
kv-seq Implementation detail. DO NOT USE. Will be private once out of alpha.
